Так как мы имеем сильного администратора, потребности к масштабированию и пока еще небольшой штат, архитектор (CTO) рассматривает вариант перехода в облачную инфраструктуру, которая позволит:
1. Быть уверенным, что инфраструктура защищена и сертифицирована ФСТЭК
2. Удовлетворяет требования ФЗ-152 по хранению и обработке (снижение трудозатрат на контроль своей инфраструктуры)
3. При определенных вводных снизить капитальные издержки компании (за счет гибкости облака)
4. Масштабироваться по необходимости
5. Развернуть дополнительные средства безопасности (Lockbox, KMS)
6. Возможность использования IAM, OAuth2 для SSO Yandex и отказ от Active Directory (можно позволить, пока штат не вырос)
7. Использовать Managed PostgreSQL, Clickhouse (компетенции CTO) при переходе от файлового режима 1C
8. Отказаться от Windows Server 2022 (импортозамещение) за счет использования Optimized Images (если 1с позволяет), либо VM на Linux
9. Можно попробовать GOST Gateway (preview) для соответствия ГОСТ Р 57580.1-2017 для мед.данных - https://yandex.cloud/ru/services/gost-gateway, (нужно подавать заявку, чтобы проверить)
10. Использовать S3 с контролем доступа для хранения файлов с автошифрованием при заливе
11. Облачное логирование
12. Managed Apache Airflow (NiFi на отдельной машине)

Также CTO предлагает найти middle-python разработчика для описания скриптов Airflow.

Учитываем правила Privacy by Design:
1. Proactive / Preventive: деидентификация на входе, автоматические проверки приватности до публикации данных.
2. Default privacy: по умолчанию храним только необходимые поля, все сырьё — в шифрованном S3, доступ через роли IAM.
3. Embedded: токенизация/маскирование встроены в пайплайны Airflow; контроль доступа на уровне сервисных ролей и сетей.
4. Full lifecycle: шифрование at-rest (SSE-KMS), in-transit (TLS), политики хранения и безопасная утилизация, аудит в Cloud Logging.
5. Visibility & Transparency: централизованный аудит пайплайнов и доступов; логирование в облаке.
6. Respect for users: соблюдаем цели обработки, retention, минимизация; собираем и учитываем согласия.
